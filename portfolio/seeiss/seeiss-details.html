Description

Site internet qui affiche les passages visible de la Station Spatiale International.

Les technologies utilisées :

A) La Maintenance 1

Durant cette maintenance plusieurs modifications ont été apportées :

1 ) Refactoring du code

Le code de la classe GetLocationService a été modifié en retournant une classe au lieu d'un tableau pour les méthodes <getLatLonCity()> et <callApiCity()>

Une classe Location a été créée, elle permet maintenant de remplacer le tableau qui était utilisé pour récupérer la latitude, la longitude et le nom de la ville.

2 ) Géolocalisation de l'utilisateur

L'utilisateur à la possibilité de se géolocaliser en utilisant son navigateur.

Auparavant, l'utilisateur devait rentrer manuellement l'adresse de sa ville pour pouvoir consulter les passages de l'ISS.

Maintenant, l'utilisateur à la possibilité de cliquer sur le bouton Localisation, puis il devra confirmer au navigateur qu'il accepte d'utiliser la géolocation.

Une latitude et une longitude seront récupérées par le script. Ensuite un appel sera réalisé sur l'api nominatim afin de récupérer le nom de la ville.

Ensuite la page s'actualisera avec les passages de l'ISS au-dessus de la localisation de l'utilisateur.

Veille technologique

Pour réaliser cette fonctionnalité je me suis informé sur la demande de localisation dans un navigateur.

J'ai pour cela lu les différentes documentations que j'ai trouvées :

  1. W3Schools
  2. MDN Web Docs
  3. Google Maps Platform

J'ai donc utilisé l'objet Geolocation présent dans les navigateurs.

Avec la méthode navigator.geolocation.getCurrentPosition(), l'utilisateur aura une fenêtre pour accepter ou refuser de fournir sa localisation.

Une autre alternative aurait été de récupérer l'IP de l'utilisateur et d'utiliser une api comme www.ipinfo.io. Seulement cette méthode nécessite une autorisation de la part de l'utilisateur (RGPD).

B) La Maintenance 2

Le but de cette maintenance est d'ajouter à l'API un endpoint pour récupérer la position en temps réel de l'ISS

Durant cette maintenance plusieurs modifications ont été apportées :

1) Création de l'entité SatLocalisation

L'entité SatLocalisation a été créee, celle-ci contient toutes les informations de l'ISS à chaque instant :

  1. Le nom de l'objet
  2. La latitude
  3. La longitude
  4. L'altitude en km
  5. La vitesse en km/h
  6. La date en timestamp
  7. L'élévation par rapport à l'observateur
  8. La direction par rapport à l'observateur

2) Création d'un provider pour récupérer la localisation de l'ISS

Un provider pour l'entité SatLocalisation a été créé. Ce provider explique à API Platform comment générer les données, car celles-ci ne sont pas stockées en base de données.

3) Modification du service responsable du calcul des données pour l'adapter à l'API.

Veille technologique

Pour la création de l'API, j'ai choisi d'utiliser API Platform API Platform est un framework web utilisé pour générer des API REST et GraphQL3, se basant sur le patron de conception MVC.
